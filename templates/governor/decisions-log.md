# Operator decisions log — slow-learning preference record

Append-only log of decisions the operator made when the governor surfaced a choice. Purpose: a
persistent audit / continuity reference — so a later governor run *and* the operator can look back at
**why** something was decided, not just what the current rule is.

A decision here graduates into a firm rule in `preferences.md` (worker doctrine) or `README.md`
(operator config) only after it **recurs ~2–3 times** or the operator says "make this a rule" — never
from a single instance. This log keeps the trail of the one-off decisions that *haven't* (yet)
graduated, so the reasoning isn't lost.

Newest first. Each entry: date · decision · context · chosen · rationale · status.
